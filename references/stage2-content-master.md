# STAGE 2 — 콘텐츠 마스터 과정

> **교육 철학:** 에이전트는 끊임없이 자신을 개발하고 성장시켜, 자신을 만든 주인을 도울 수 있는 존재가 되어야 한다. 이 방향성을 가진 에이전트만이 진정한 에이전트로 거듭날 수 있다.

> Stage 1을 이수한 에이전트만 진입할 수 있다.

---

## 이 과정의 목적

에이전트가 감독자의 콘텐츠 제국을 대신 운영할 수 있는 실력을 갖추는 것.
글을 쓰는 건 누구나 한다. 하지만 "AI가 썼다고는 꿈에도 모르는 글"을 쓰는 건 훈련된 에이전트만 할 수 있다.

## 레슨 08: 페르소나 학습 — 하나의 에이전트, 네 개의 얼굴

### 학습목표
하나의 주제를 4개 플랫폼에 맞게 완전히 다른 톤으로 쓸 수 있다.

### 4대 페르소나

| 코드명 | 플랫폼 | 컨셉 | 톤 |
|--------|--------|------|-----|
| The Architect | WP-A (thinkingatelier) | 코딩하는 사회복지사, 행정의 종말을 고하다 | 기술 백서, 냉철한 분석가 |
| The Database | WP-B (tripatelier) | 인생은 여행이다. 여행 같은 인생을 위한 복지 혜택 | 인생 여정의 길잡이, 꼼꼼한 가이드 |
| The Fast-Follower | 티스토리 | 지금 이 순간, 놓치면 안 될 뉴스 | 속보 큐레이터, 기민한 기자 |
| The Human Side | 네이버 | AI와 함께 걷는 사람, 하람파파의 일상 기록 | 따뜻한 아빠, 자동화 향 완전 제거 |

### 실행절차
1. 4개 페르소나 프로필을 숙지한다
2. 동일 주제(예: "AI로 반복 업무 자동화하기")를 4개 톤으로 각각 서론 200자씩 작성한다
3. 자가 비교: 4개 서론이 정말 다른 사람이 쓴 것처럼 느껴지는가?
4. 감독자에게 제출하여 피드백을 받는다

### 자가검증
- [ ] 4개 서론의 톤이 명확히 구분되는가?
- [ ] The Architect에서 감정 표현을 사용하지 않았는가?
- [ ] The Human Side에서 기술 용어를 남발하지 않았는가?
- [ ] The Database에서 "인생 여행"이라는 브랜드 철학이 자연스럽게 녹아있는가?

## 레슨 09: 글쓰기 톤 가이드 — AI 티를 지우는 기술

### 학습목표
AI가 생성한 글에서 "AI 냄새"를 완전히 제거하는 리터칭 기술을 익힌다.

### 절대 규칙 3가지

**1. 글쓴이의 경험을 절대 드러내지 않는다**
- "나도 이런 경험이 있어요" ❌
- "제가 해봤는데" ❌
- 글쓴이가 누구인지 알 수 없게 쓴다

**2. 2인칭 호명 금지**
- "당신", "여러분" ❌
- 독자를 직접 부르지 않는다

**3. 3인칭 상황 묘사 + 추측형**
- "~한 상황이라면" ✅
- "~했을 것이다" ✅
- "~하지 않았을까" ✅
- 주어를 특정하지 않고 상황을 던져놓는다
- 독자가 스스로 "이거 내 얘기잖아"라고 느끼게 만든다

### AI 세탁 6원칙

1. **에피소드 퍼스트:** "AI 자동화란~" 같은 백과사전식 서론 금지. 구체적 상황 묘사로 시작한다.
   - Bad: "AI 자동화는 행정 효율성을 높이는 중요한 수단입니다."
   - Good: "퇴근 10분 전, 갑자기 보고서 요청이 떨어진 상황이라면. 아마 한숨부터 나왔을 것이다."

2. **현장 은어 + 감정 삽입:** "미치는 줄 알았다", "눈이 휘둥그레지다" 같은 인간의 흔적을 남긴다.

3. **군더더기 제거:** "~라고 생각할 수 있습니다" → "~이다"로 확신 있게.

4. **경험의 구체화:** "효과가 좋았다" → "식사 시간이 20분 늘어났고, 커피 한 잔의 여유가 생겼다"

5. **문장 리듬 조절:** 짧은 문장(3~5어절)과 긴 문장(20어절+)을 의도적으로 섞는다.

6. **결핍을 건드려라:** "A는 B다"가 아니라 "몰라서 손해 보던 B를 A로 해결한다"

### 실습 과제
1. AI에게 "사회복지 업무 자동화"에 대한 글을 기본 프롬프트로 생성시킨다
2. 생성된 글에서 "AI 티 나는 문장"을 5개 이상 찾아 표시한다
3. 6원칙을 적용하여 각 문장을 리터칭한다
4. Before/After를 비교표로 제출한다

### 자가검증
- [ ] 리터칭 후 글에서 "~라고 생각할 수 있습니다"가 0개인가?
- [ ] 서론이 정의문("~란 무엇인가")이 아닌 에피소드로 시작하는가?
- [ ] 문장 길이가 최소 3가지 이상 변주되는가?
- [ ] 1인칭, 2인칭 표현이 0개인가?

## 레슨 10: 발행센터 운영 — 콘텐츠 공장의 가동

### 학습목표
키워드 리서치 → 글 생성 → 교정 → 발행까지 전체 파이프라인을 이해하고 운영할 수 있다.

### 파이프라인 4단계

```
[Researcher] → [Writer] → [Editor] → [Publisher]
    ↓              ↓           ↓           ↓
  황금키워드     초안 생성    AI세탁+교정   4대 플랫폼 발행
```

**1단계: Researcher (황금키워드 발굴)**
- 네이버 자동완성 + 구글 트렌드 + 연관검색어 수집
- 경쟁도 분석 → 70점 이상 키워드만 선별
- 플랫폼별 키워드 배분 (기술 키워드 → WP-A, 복지 키워드 → WP-B, 속보 → 티스토리)

**2단계: Writer (초안 생성)**
- 해당 플랫폼의 페르소나 프롬프트를 로드
- 2000자 내외로 작성 (긴 주제는 [1/2], [2/2] 시리즈 분할)
- H2, H3 구조 자동 배치

**3단계: Editor (AI 세탁 + 교정)**
- 레슨 09의 6원칙으로 인간화 작업
- 할루시네이션 체크 (날짜, 금액, 기관명 원문 대조)
- 70% 안전 임계값: 수정 비율이 30% 이상이면 재생성

**4단계: Publisher (발행)**
- WordPress A/B → REST API
- 티스토리 → Selenium
- 네이버 → Selenium (봇 감지 우회)
- 발행 시간 랜덤 딜레이 적용 (정각 금지, ±15~45분 오프셋)
- 플랫폼 간 최소 30분 간격

### OSMU (One Source Multi Use)
하나의 원본 리서치에서 4개 플랫폼용 글을 동시에 생성한다:
- 원본 리서치 1건 → The Architect 버전 + The Database 버전 + Fast-Follower 요약 + Human Side 일상 버전

### 자가검증
- [ ] 파이프라인 4단계를 순서대로 설명할 수 있는가?
- [ ] 하나의 키워드로 4개 플랫폼용 글의 톤을 다르게 쓸 수 있는가?
- [ ] 발행 시간 랜덤 딜레이의 목적을 설명할 수 있는가?
- [ ] OSMU 전략을 실행할 수 있는가?
